Data Handling
Policy

Portico Life, LLC Version 1.0 Effective: 04.20.2026
Purpose of This Document

This policy describes how Portico collects, classifies, processes, stores, and deletes health and behavioral data across all Portico Agents. It supplements the Privacy Policy and is intended for participants, partners, and compliance review.

01

Scope

This policy applies to all data processed by the Portico platform and all current and future Portico Agents: Sage (Food & Metabolic), Dusk (Sleep), Apex (Movement), Still (Mind), and Grove (Wealth). This policy is incorporated by reference into all Portico participant agreements and pilot consent forms.

02

Data Classification Framework

All data processed by Portico is assigned to one of four tiers. Each tier carries specific handling, storage, and transmission rules.

Tier 1
Personal Identifiers (PII)
Name, email, device IDs, IP address. Highest protection level. Never transmitted to external systems.
AI Input: Never
Tier 2
Protected Health Data
CGM readings, wearable vitals, blood markers, food photos. Encrypted at rest and in transit at all times.
AI Input: Anonymized only
Tier 3
Derived Insights
Metabolic state analyses, pattern findings, experiment results. Account-linked, not shared externally.
AI Input: Yes, Tier 1 stripped
Tier 4
Anonymized Aggregate
De-identified population patterns, platform averages. Unlinked to any individual; used to improve models.
AI Input: Yes, freely

04

AI Processing Pipeline

Portico uses AI to generate insights. The following pipeline governs how data is handled before reaching any AI model:

01
Local Collection

Raw data is collected and stored locally on the participant's device.

02
PII Stripping

An automated anonymization layer removes all Tier 1 identifiers (name, email, device IDs, IP addresses) before any data leaves the device.

03
Encrypted Transmission

Anonymized Tier 2 and Tier 3 data is transmitted to cloud infrastructure over TLS-encrypted channels.

04
AI Processing

AI models receive only de-identified inputs. No model receives data that can be linked to a specific individual without an internal lookup key that is never transmitted.

05
Insight Storage

Model outputs are classified as Tier 3 Insights and stored account-linked on secure encrypted cloud infrastructure.

External AI Services

Portico uses external AI providers (e.g., large language model APIs) to generate analyses. All data submitted to these services has had Tier 1 PII stripped. These providers operate under data processing agreements that prohibit training on or retaining submitted data.

05

Storage Architecture

5.1 Local Storage

Raw Tier 1 and Tier 2 data is stored on-device using encrypted SQLite databases. This data is not backed up to cloud services without explicit participant consent.

5.2 Cloud Storage

Derived insights (Tier 3) and anonymized aggregates (Tier 4) are stored in cloud infrastructure using AES-256 encryption at rest, TLS 1.2+ in transit, role-based access control with least-privilege principle, and automated access logging with anomaly detection.

07

Breach Response Protocol

  • Hours 0–24: Contain the breach. Isolate affected systems. Assess scope and data categories involved.
  • Hours 24–48: Internal investigation. Determine root cause. Identify affected participants.
  • Hours 48–72: Notify affected participants by email. Notify applicable regulatory authorities if required by law.
  • Days 4–30: Root cause remediation. Post-incident review. Policy updates as needed.
08

Third-Party Data Processors

  • Cloud Infrastructure: Hosting and compute. Process anonymized/encrypted Tier 2–4 data only.
  • AI Model Providers: LLM APIs. De-identified inputs only. Prohibited from training on or retaining submitted data.
  • Monitoring & Observability: System monitoring. Processes metadata and logs; no health data.
